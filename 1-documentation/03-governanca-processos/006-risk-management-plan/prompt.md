# Prompts para Risk Management Plan

## Prompt para Plano de Gerenciamento de Riscos Completo

```
Você é um especialista em gestão de riscos com experiência em diversos tipos de projetos. Preciso da sua ajuda para criar um Plano de Gerenciamento de Riscos completo para o projeto [nome do projeto].

Contexto do projeto:
- Descrição: [breve descrição do projeto]
- Tamanho/complexidade: [pequeno, médio, grande, complexo]
- Duração estimada: [período]
- Orçamento aproximado: [valor ou faixa]
- Setor: [área de atuação do projeto]
- Restrições críticas: [principais restrições]

Por favor, crie um Plano de Gerenciamento de Riscos detalhado que inclua:

1. Metodologia de gerenciamento de riscos
2. Papéis e responsabilidades na gestão de riscos
3. Categorias de risco relevantes para este projeto
4. Definições de probabilidade e impacto
5. Matriz de probabilidade e impacto
6. Processos de identificação, análise, resposta e monitoramento de riscos
7. Protocolos de comunicação e relatório
8. Cronograma de atividades de gerenciamento de riscos
9. Template de registro de riscos

Para cada seção, forneça conteúdo específico e adaptado às particularidades deste projeto.
```

## Prompt para Metodologia de Gerenciamento de Riscos

```
Como consultor de gestão de riscos, preciso definir uma metodologia de gerenciamento de riscos para o projeto [nome do projeto].

Contexto do projeto:
- Tipo de projeto: [tipo]
- Maturidade da organização em gestão de riscos: [baixa, média, alta]
- Recursos disponíveis: [limitados, adequados, abundantes]
- Complexidade do projeto: [baixa, média, alta]

Por favor, ajude-me a desenvolver:

1. Uma abordagem geral de gestão de riscos adequada a este contexto:
   - Framework geral recomendado (PMI, ISO 31000, personalizado)
   - Nível de formalidade e rigor
   - Adaptações necessárias para este projeto específico
   - Justificativa para as escolhas recomendadas

2. Ferramentas e técnicas específicas para:
   - Identificação de riscos (brainstorming, checklists, entrevistas, etc.)
   - Análise qualitativa de riscos
   - Análise quantitativa (se aplicável)
   - Planejamento de respostas
   - Monitoramento e controle

3. Fontes de dados e informações:
   - Internas (projetos anteriores, especialistas, etc.)
   - Externas (benchmarks, estudos da indústria, etc.)
   - Abordagem para coleta e validação

4. Níveis de precisão exigidos:
   - Detalhamento do registro de riscos
   - Precisão das estimativas de probabilidade
   - Precisão das estimativas de impacto
   - Frequência de reavaliação

Forneça exemplos concretos de como cada elemento da metodologia seria aplicado neste projeto específico.
```

## Prompt para Estrutura de Decomposição de Riscos (RBS)

```
Como especialista em análise de riscos, preciso desenvolver uma Estrutura de Decomposição de Riscos (RBS) abrangente para o projeto [nome do projeto].

Contexto do projeto:
- Setor/indústria: [setor]
- Principais componentes técnicos: [componentes]
- Stakeholders principais: [stakeholders]
- Ambientes de operação: [ambientes]

Por favor, ajude-me a criar:

1. Uma estrutura hierárquica de categorias de risco adaptada a este projeto:
   - Categorias de primeiro nível (técnicos, externos, organizacionais, gerenciais)
   - Subcategorias detalhadas para cada categoria principal
   - Definições claras de cada categoria e subcategoria
   - Exemplos específicos de riscos em cada categoria

2. Para as categorias técnicas, decompor por:
   - Áreas técnicas específicas do projeto
   - Complexidade e novidade
   - Interfaces e dependências
   - Qualidade e desempenho

3. Para riscos externos, categorizar por:
   - Regulatórios e de conformidade
   - Mercado e economia
   - Ambientais e climáticos
   - Políticos e sociais
   - Contratuais e legais

4. Para riscos organizacionais, incluir:
   - Recursos e priorização
   - Financiamento e orçamento
   - Governança e tomada de decisão
   - Comunicação e cultura

5. Para riscos gerenciais, subdividir em:
   - Estimativas e planejamento
   - Controle e monitoramento
   - Comunicação e relatórios
   - Gestão de stakeholders

Forneça um diagrama textual da RBS completa, com descrições detalhadas de cada componente e como ele se aplica especificamente a este projeto.
```

## Prompt para Definições de Probabilidade e Impacto

```
Como analista de riscos, preciso estabelecer definições claras de probabilidade e impacto para o projeto [nome do projeto].

Contexto do projeto:
- Objetivos principais: [objetivos]
- Restrições críticas: [restrições]
- Tolerância a risco dos stakeholders: [baixa, média, alta]
- Duração e complexidade: [detalhes relevantes]

Por favor, ajude-me a criar:

1. Escalas detalhadas de probabilidade:
   - Escala de 5 níveis (muito baixa a muito alta)
   - Definições qualitativas de cada nível
   - Intervalos quantitativos correspondentes (percentuais)
   - Exemplos práticos para este projeto específico

2. Escalas de impacto para cada objetivo do projeto:
   - Impacto em custo
   - Impacto em cronograma
   - Impacto em escopo/qualidade
   - Impacto em outros objetivos específicos [listar]

3. Para cada tipo de impacto, definir:
   - Escala de 5 níveis (muito baixo a muito alto)
   - Descrições qualitativas de cada nível
   - Métricas quantitativas quando possível
   - Exemplos concretos relevantes para este projeto

4. Matriz de probabilidade e impacto:
   - Layout visual da matriz 5x5
   - Definição de zonas de risco (baixo, moderado, alto, extremo)
   - Códigos de cores ou numeração
   - Thresholds para diferentes níveis de resposta

5. Regras para avaliação de risco geral:
   - Método para combinar múltiplos tipos de impacto
   - Ponderação de diferentes objetivos (se aplicável)
   - Regras para resolução de ambiguidades
   - Frequência de reavaliação

Inclua exemplos práticos de como avaliar riscos específicos para este projeto usando estas definições.
```

## Prompt para Estratégias de Resposta a Riscos

```
Como gerente de riscos, preciso desenvolver estratégias de resposta a riscos para o projeto [nome do projeto].

Contexto:
- Principais categorias de risco: [categorias]
- Restrições de recursos: [restrições]
- Nível de autoridade da equipe: [detalhes relevantes]
- Tolerância a risco: [baixa, média, alta]

Por favor, ajude-me a criar:

1. Um framework para estratégias de resposta a ameaças:
   - Evitar: critérios e abordagens
   - Transferir: mecanismos e considerações
   - Mitigar: técnicas e níveis de mitigação
   - Aceitar: critérios para aceitação ativa vs. passiva
   - Exemplos específicos para cada estratégia

2. Um framework para estratégias de resposta a oportunidades:
   - Explorar: métodos para garantir a realização
   - Compartilhar: abordagens para parcerias
   - Melhorar: técnicas para aumentar probabilidade/impacto
   - Aceitar: quando aproveitar se disponível
   - Exemplos específicos para cada estratégia

3. Critérios de seleção de estratégias:
   - Análise custo-benefício
   - Avaliação de riscos secundários
   - Considerações de timing
   - Alinhamento com objetivos do projeto
   - Matriz de decisão para seleção

4. Processo para desenvolvimento de planos de resposta:
   - Template para documentação de respostas
   - Níveis de detalhe necessários
   - Responsabilidades e aprovações
   - Integração com o cronograma e orçamento
   - Gatilhos para implementação

5. Planejamento de contingência:
   - Critérios para desenvolver planos de contingência
   - Estrutura de planos de fallback
   - Reservas de contingência vs. reservas de gestão
   - Procedimentos de ativação

Forneça exemplos de estratégias de resposta para 3-5 riscos hipotéticos mas realistas para este projeto específico.
```

## Prompt para Monitoramento e Controle de Riscos

```
Como gerente de projeto, preciso estabelecer um sistema de monitoramento e controle de riscos para o projeto [nome do projeto].

Contexto:
- Duração do projeto: [duração]
- Tamanho da equipe: [tamanho]
- Frequência de reporting: [frequência]
- Ferramentas disponíveis: [ferramentas]

Por favor, ajude-me a desenvolver:

1. Um processo abrangente de monitoramento de riscos:
   - Frequência de revisão para diferentes categorias de risco
   - Métodos de acompanhamento de gatilhos e indicadores
   - Técnicas para identificar novos riscos
   - Protocolos para reavaliação de riscos existentes
   - Integração com outros processos de monitoramento

2. Um sistema de rastreamento e documentação:
   - Estrutura do registro de riscos
   - Métricas para acompanhamento
   - Processo de atualização
   - Histórico de alterações
   - Rastreamento de ações de resposta

3. Protocolos de comunicação e relatório:
   - Relatórios periódicos (conteúdo, formato, frequência)
   - Relatórios de exceção (gatilhos, urgência)
   - Dashboards e visualizações
   - Destinatários por tipo de relatório
   - Feedback e ações

4. Reuniões e revisões de risco:
   - Tipos de reuniões (propósito, participantes)
   - Agendas padronizadas
   - Inputs necessários
   - Outputs e follow-up
   - Integração com outras reuniões do projeto

5. Processo para implementação de respostas e planos de contingência:
   - Aprovações necessárias
   - Documentação de implementação
   - Verificação de eficácia
   - Ajustes e ações corretivas
   - Lições aprendidas

6. Métricas de eficácia do gerenciamento de riscos:
   - KPIs de processo
   - Medidas de resultado
   - Benchmarks
   - Processo de melhoria contínua

Forneça templates e exemplos práticos para cada componente do sistema adaptados ao contexto específico deste projeto.
```
