# Prompts para RACI Matrix

## Prompt para Matriz RACI Completa

```
Você é um especialista em governança de projetos com experiência em definir claramente papéis e responsabilidades. Preciso da sua ajuda para criar uma Matriz RACI completa para o projeto [nome do projeto].

Contexto do projeto:
- Descrição: [breve descrição do projeto]
- Papéis principais: [listar papéis/funções-chave no projeto]
- Principais áreas de atividade: [listar principais grupos de atividades]
- Metodologia: [Agile, Waterfall, híbrida, etc.]

Por favor, crie uma Matriz RACI detalhada que:

1. Liste todas as atividades importantes do projeto organizadas por fase/área
2. Atribua claramente designações RACI (Responsável, Aprovador, Consultado, Informado) para cada papel
3. Inclua notas explicativas para situações complexas ou ambíguas
4. Forneça diretrizes de uso e interpretação da matriz

Além disso, inclua recomendações para garantir uma implementação eficaz da matriz RACI neste projeto específico.
```

## Prompt para Identificação de Atividades e Papéis

```
Como gerente de projeto, preciso identificar todas as atividades relevantes e papéis para a criação de uma Matriz RACI para o projeto [nome do projeto].

Contexto do projeto:
- Tipo de projeto: [tipo]
- Metodologia: [metodologia]
- Principais deliverables: [listar deliverables-chave]
- Estrutura organizacional: [descrição básica]

Por favor, ajude-me a:

1. Identificar e categorizar todas as atividades significativas do projeto:
   - Atividades de gerenciamento (governança, reporting, etc.)
   - Atividades técnicas/de desenvolvimento
   - Atividades de qualidade e testes
   - Atividades de implantação e operação
   - Atividades de documentação e treinamento
   - [Outras categorias relevantes]

   Para cada categoria, liste 5-10 atividades específicas que devem aparecer na matriz.

2. Identificar todos os papéis que devem ser incluídos na matriz:
   - Papéis de liderança e gerenciamento
   - Papéis técnicos e de desenvolvimento
   - Papéis de stakeholders externos
   - Papéis de suporte e operações
   - [Outros papéis relevantes]

   Para cada papel, forneça uma breve descrição de sua função no projeto.

3. Sugerir o nível de detalhamento apropriado:
   - Análise de granularidade ideal para as atividades
   - Recomendação se papéis devem ser individuais ou por equipe/departamento
   - Como lidar com atividades recorrentes vs. únicas

4. Recomendações para a estrutura da matriz:
   - Formato mais adequado para este projeto
   - Agrupamento lógico das atividades
   - Ordenação cronológica ou funcional
```

## Prompt para Atribuição de Responsabilidades RACI

```
Como facilitador de workshops de RACI, preciso de diretrizes para atribuir corretamente as responsabilidades RACI para o projeto [nome do projeto].

Já identificamos as seguintes atividades e papéis principais:
- Atividades: [listar atividades-chave]
- Papéis: [listar papéis-chave]

Por favor, ajude-me com:

1. Diretrizes claras para atribuir cada tipo de responsabilidade:
   - Como identificar quem deve ser o "Responsável" (R) para cada atividade
   - Critérios para designar o "Aprovador" (A)
   - Como determinar quem deve ser "Consultado" (C)
   - Quando incluir alguém como "Informado" (I)

2. Regras e princípios para uma matriz RACI equilibrada:
   - Número ideal de "R"s por atividade
   - Como garantir que cada atividade tenha exatamente um "A"
   - Equilíbrio entre "C"s e "I"s
   - Como evitar sobrecarga de responsabilidades para certos papéis

3. Exemplos de atribuições típicas para diferentes tipos de atividades em projetos similares:
   - Padrões comuns para atividades de governança
   - Padrões comuns para atividades técnicas
   - Padrões comuns para atividades de qualidade
   - [Outros padrões relevantes]

4. Processo para resolver conflitos e ambiguidades:
   - Como lidar com sobreposições de responsabilidade
   - Processo para resolver disputas sobre atribuições
   - Como tratar atividades que parecem não ter um responsável natural

5. Estratégias para facilitar um workshop de RACI produtivo:
   - Técnicas de facilitação
   - Como obter consenso
   - Como documentar decisões e justificativas
```

## Prompt para Análise e Otimização da Matriz RACI

```
Como consultor de eficiência organizacional, preciso analisar e otimizar uma matriz RACI existente para o projeto [nome do projeto].

A matriz atual tem as seguintes características:
- Número de atividades: [quantidade]
- Número de papéis: [quantidade]
- Principais problemas identificados: [problemas, se conhecidos]

Por favor, ajude-me a:

1. Desenvolver critérios para analisar a qualidade da matriz RACI:
   - Indicadores de matriz desequilibrada
   - Sinais de sobrecarga de responsabilidades
   - Sinais de lacunas em responsabilidades
   - Evidências de ambiguidades

2. Realizar uma análise da matriz:
   - Análise de carga horizontal (número de designações por atividade)
   - Análise de carga vertical (número de designações por papel)
   - Verificação de "órfãos" (atividades sem responsáveis claros)
   - Verificação de "super-heróis" (papéis com excesso de responsabilidades)

3. Recomendar otimizações específicas:
   - Redistribuição de responsabilidades sobrecarregadas
   - Preenchimento de lacunas identificadas
   - Consolidação de atividades muito fragmentadas
   - Esclarecimento de ambiguidades nas designações

4. Sugerir um processo de revisão contínua:
   - Frequência de revisão recomendada
   - Gatilhos para revisão extraordinária
   - Processo para implementar mudanças
   - Métricas para avaliar a eficácia das mudanças
```

## Prompt para Implementação e Comunicação da Matriz RACI

```
Como especialista em gestão de mudanças, preciso desenvolver um plano para implementar e comunicar efetivamente a matriz RACI no projeto [nome do projeto].

Contexto:
- Tamanho da equipe: [tamanho]
- Distribuição geográfica: [local, distribuído, híbrido]
- Familiaridade prévia com RACI: [alta, média, baixa]
- Cultura organizacional: [descrição relevante]

Por favor, ajude-me a criar:

1. Um plano de lançamento da matriz RACI:
   - Preparação necessária antes do lançamento
   - Sequência de atividades para implementação
   - Timeline recomendado
   - Pré-requisitos para sucesso

2. Estratégia de comunicação:
   - Mensagens-chave para diferentes audiências
   - Canais de comunicação recomendados
   - Materiais de suporte necessários
   - Frequência de comunicações

3. Plano de treinamento:
   - Conteúdo recomendado para treinamento
   - Formato e duração
   - Público-alvo prioritário
   - Avaliação de eficácia

4. Estratégias para adoção e conformidade:
   - Como incentivar a adoção
   - Como monitorar o uso da matriz
   - Como lidar com resistência
   - Como medir sucesso

5. Plano para manutenção contínua:
   - Processo de atualização da matriz
   - Responsáveis pela manutenção
   - Integração com outros processos de governança
   - Estratégia de melhoria contínua

Forneça exemplos concretos e templates de comunicação adaptados ao contexto específico deste projeto.
```
