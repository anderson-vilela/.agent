# Prompts para Governance Framework

## Prompt para Estrutura Completa do Governance Framework

```
Você é um especialista em governança de projetos com experiência em estabelecer estruturas organizacionais e sistemas de tomada de decisão. Preciso da sua ajuda para criar um Governance Framework completo para o projeto [nome do projeto].

Contexto do projeto:
- Descrição: [breve descrição do projeto]
- Tamanho/complexidade: [pequeno, médio, grande, complexo]
- Duração estimada: [período]
- Stakeholders principais: [lista de stakeholders-chave]
- Contexto organizacional: [estrutura da organização, cultura, restrições]

Por favor, crie um Governance Framework detalhado que inclua:

1. Estrutura organizacional do projeto
2. Mecanismos de tomada de decisão
3. Papéis e responsabilidades detalhados
4. Políticas e princípios de governança
5. Estratégias de gestão de stakeholders
6. Mecanismos de supervisão e controle
7. Processo de gestão de mudanças na governança

Para cada seção, inclua exemplos práticos, modelos e templates quando aplicável, e recomendações específicas para este projeto.
```

## Prompt para Estrutura Organizacional do Projeto

```
Como consultor de governança, preciso definir a estrutura organizacional para o projeto [nome do projeto].

Detalhes relevantes:
- Escopo do projeto: [breve descrição do escopo]
- Tamanho da equipe: [número aproximado]
- Estrutura organizacional existente: [descrição da organização atual]
- Restrições ou requisitos específicos: [considerações especiais]
- Metodologia do projeto: [Agile, Waterfall, híbrida]

Por favor, ajude-me a desenvolver:

1. Organograma completo do projeto, incluindo:
   - Hierarquia clara
   - Linhas de reporte
   - Grupos funcionais
   - Equipes interfuncionais

2. Comitês e grupos de decisão:
   - Comitê diretor (steering committee)
   - Grupos de trabalho técnico
   - Comunidades de prática
   - Frequency e formato de reuniões

3. Interfaces com a organização mais ampla:
   - Pontos de conexão com estruturas existentes
   - Escalação e reportes para níveis superiores
   - Integração com outras iniciativas

4. Justificativa para as escolhas estruturais:
   - Alinhamento com objetivos do projeto
   - Considerações culturais
   - Otimização para a tomada de decisão

Forneça um diagrama textual da estrutura proposta, com descrições detalhadas de cada componente e as razões para as escolhas feitas.
```

## Prompt para Mecanismos de Tomada de Decisão

```
Como especialista em governança, preciso definir mecanismos eficientes de tomada de decisão para o projeto [nome do projeto].

Contexto do projeto:
- Tipo de projeto: [tipo]
- Criticidade para o negócio: [baixa, média, alta]
- Restrições de tempo: [urgência do projeto]
- Cultura organizacional: [descrição da cultura de decisão]

Por favor, ajude-me a estabelecer:

1. Categorização de decisões:
   - Tipos de decisões no projeto (estratégicas, táticas, operacionais)
   - Níveis de impacto e criticidade
   - Timebox para diferentes tipos de decisão

2. Processo de tomada de decisão:
   - Etapas formais para diferentes categorias de decisão
   - Métodos de análise e avaliação
   - Ferramentas de suporte à decisão
   - Processo para decisões emergenciais

3. Autoridade decisória:
   - Matriz de autoridade por tipo de decisão
   - Limites financeiros/escopo para diferentes níveis
   - Mecanismo de delegação
   - Decisões que requerem consenso vs. autoridade individual

4. Documentação e comunicação:
   - Templates para registro de decisões
   - Processo para comunicar decisões
   - Rastreabilidade de decisões
   - Repositório de decisões-chave

5. Resolução de conflitos:
   - Processo para impasses
   - Mecanismos de escalação
   - Mediação e arbitragem

Forneça exemplos concretos e templates para cada componente, considerando a natureza específica do projeto.
```

## Prompt para Papéis e Responsabilidades

```
Como especialista em estruturas organizacionais, preciso definir claramente os papéis e responsabilidades para o projeto [nome do projeto].

O projeto envolve os seguintes grupos de stakeholders:
- [Grupo 1] (ex: executivos)
- [Grupo 2] (ex: gerência)
- [Grupo 3] (ex: equipe técnica)
- [Outros grupos relevantes]

Por favor, ajude-me a desenvolver:

1. Descrições detalhadas dos papéis-chave:
   - Patrocinador do Projeto
   - Gerente de Projeto
   - Product Owner/Business Analyst
   - Arquiteto/Líder Técnico
   - [Outros papéis específicos do projeto]

Para cada papel, inclua:
   - Objetivos e escopo de responsabilidade
   - Autoridade e limites de decisão
   - Qualificações e competências requeridas
   - Tempo de dedicação esperado
   - Principais deliverables e atividades
   - Interfaces com outros papéis

2. Matriz de responsabilidades de alto nível:
   - Áreas principais de responsabilidade
   - Papéis envolvidos em cada área
   - Tipos de envolvimento (mais detalhado que RACI simples)
   - Interdependências críticas

3. Mecanismos de delegação:
   - Processo formal para delegação
   - Limites de delegação
   - Supervisão de responsabilidades delegadas
   - Revogação de delegações

4. Plano de substituição e continuidade:
   - Backup para papéis críticos
   - Processo de transição
   - Gestão de conhecimento para continuidade

Forneça exemplos concretos e templates de descrições de papéis que sejam específicos para este tipo de projeto.
```

## Prompt para Políticas e Princípios de Governança

```
Como consultor de governança, preciso estabelecer um conjunto de políticas e princípios de governança para o projeto [nome do projeto].

Contexto relevante:
- Tipo de organização: [empresa, governo, sem fins lucrativos, etc.]
- Políticas organizacionais existentes: [políticas que devem ser consideradas]
- Requisitos regulatórios: [regulamentações aplicáveis]
- Valores organizacionais: [valores declarados da organização]

Por favor, ajude-me a desenvolver:

1. Princípios guia de governança:
   - 5-7 princípios fundamentais que orientarão a governança do projeto
   - Justificativa para cada princípio
   - Como os princípios se alinham aos valores organizacionais
   - Exemplos de aplicação prática

2. Políticas específicas de governança para:
   - Transparência e comunicação
   - Gestão de conflitos de interesse
   - Gerenciamento de mudanças
   - Aprovações e autorizações
   - Gestão de qualidade
   - Conformidade e auditoria
   - Gestão de performance
   - [Outras áreas relevantes]

3. Alinhamento com políticas organizacionais:
   - Como as políticas do projeto se conectam com políticas corporativas
   - Adaptações necessárias
   - Resolução de conflitos entre políticas

4. Processo para exceções:
   - Critérios para considerar exceções
   - Processo de aprovação de exceções
   - Documentação e monitoramento de exceções
   - Revisão periódica de exceções concedidas

Forneça exemplos concretos de como cada política deve ser aplicada e templates para documentação relacionada.
```
