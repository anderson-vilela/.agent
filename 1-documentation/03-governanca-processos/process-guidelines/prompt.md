# Prompts para Process Guidelines

## Prompt para Estrutura Completa de Process Guidelines

```
Você é um especialista em metodologias de desenvolvimento e processos de projeto com vasta experiência em estabelecer diretrizes eficientes. Preciso da sua ajuda para criar Process Guidelines abrangentes para um projeto de [tipo de projeto].

Contexto do projeto:
- Descrição: [breve descrição do projeto]
- Tamanho da equipe: [número de pessoas e composição]
- Duração estimada: [tempo previsto]
- Abordagem metodológica preferida: [Agile, Waterfall, híbrida, etc.]
- Características especiais: [distribuição geográfica, requisitos regulatórios, etc.]

Por favor, crie Process Guidelines detalhadas que incluam:

1. Definição da metodologia de desenvolvimento
2. Fluxos de trabalho completos para diferentes tipos de atividades
3. Práticas de engenharia a serem seguidas
4. Processo de gestão de artefatos
5. Ferramentas e ambientes a serem utilizados
6. Sistema de métricas e monitoramento
7. Mecanismos de conformidade e tratamento de exceções

Para cada seção, inclua justificativas para as escolhas feitas e orientações práticas para implementação.
```

## Prompt para Metodologia de Desenvolvimento

```
Como metodologista ágil/especialista em processos, preciso definir a metodologia de desenvolvimento para um projeto de [tipo de projeto].

Considerando as seguintes características:
- Tamanho da equipe: [tamanho]
- Complexidade do projeto: [baixa, média, alta]
- Familiaridade da equipe com diferentes metodologias: [descrever experiência]
- Necessidades de documentação e conformidade: [requisitos específicos]
- Envolvimento do cliente: [nível de envolvimento esperado]

Por favor, ajude-me a:

1. Selecionar a metodologia mais adequada (ou combinação de metodologias)
2. Adaptar a metodologia às necessidades específicas do projeto
3. Definir:
   - Ciclos de desenvolvimento (duração, cadência)
   - Cerimônias e eventos (quais, frequência, participantes, objetivos)
   - Artefatos metodológicos (backlog, quadros Kanban, etc.)
   - Papéis e responsabilidades metodológicas
   - Sistema de priorização
   - Métricas de acompanhamento metodológico

4. Elaborar um plano de implementação da metodologia, incluindo:
   - Treinamento necessário
   - Ajustes graduais
   - Gerenciamento de resistências
   - Avaliação e refinamento contínuo
```

## Prompt para Fluxos de Trabalho

```
Como especialista em processos, preciso definir fluxos de trabalho claros para as diferentes atividades do projeto [nome do projeto].

O projeto envolve os seguintes tipos de atividades:
- [Atividade 1] (ex: desenvolvimento de novas funcionalidades)
- [Atividade 2] (ex: correção de bugs)
- [Atividade 3] (ex: melhorias de performance)
- [Outros tipos relevantes de atividades]

Para cada tipo de atividade, ajude-me a criar:

1. Um fluxo de trabalho detalhado que inclua:
   - Estados possíveis (ex: To Do, In Progress, In Review, Done)
   - Critérios de transição entre estados
   - Pontos de verificação e aprovação
   - Responsáveis em cada etapa
   - Tempo esperado em cada estado
   - Políticas de escalação

2. Diretrizes para:
   - Priorização dentro do fluxo
   - Limites de work-in-progress (WIP)
   - Tratamento de bloqueios
   - Gestão de dependências

3. Templates de documentação necessários em cada etapa

4. Métricas de acompanhamento específicas para o fluxo

Descreva os fluxos em formato textual detalhado e sugira como eles podem ser visualizados (Kanban, diagrama de fluxo, etc.).
```

## Prompt para Práticas de Engenharia

```
Como líder técnico/arquiteto, preciso estabelecer práticas de engenharia consistentes para o projeto [nome do projeto], que utiliza as seguintes tecnologias:
- [Tecnologias principais]
- [Frameworks]
- [Linguagens de programação]

Por favor, ajude-me a definir diretrizes claras para as seguintes áreas:

1. Padrões de Codificação:
   - Convenções de nomenclatura
   - Organização de código e estrutura de projeto
   - Formatação e estilo
   - Comentários e documentação de código
   - Tratamento de erros e logging

2. Controle de Versão:
   - Estratégia de branching
   - Convenções de commit
   - Processo de pull/merge request
   - Política de tags e releases

3. Revisão de Código:
   - Critérios para submissão
   - Checklist de revisão
   - Processo de feedback
   - Critérios de aprovação

4. Testes:
   - Níveis de teste (unitário, integração, etc.)
   - Cobertura mínima exigida
   - Automação de testes
   - Testes de performance/segurança

5. Integração e Entrega Contínua:
   - Pipeline CI/CD
   - Ambientes de teste/homologação
   - Processo de deployment
   - Estratégias de feature flags

Forneça exemplos práticos e justificativas para cada recomendação, considerando o contexto específico do projeto.
```

## Prompt para Gestão de Artefatos

```
Como gerente de configuração/documentação, preciso estabelecer um sistema de gestão de artefatos para o projeto [nome do projeto].

Os principais tipos de artefatos no projeto incluem:
- [Tipo de artefato 1] (ex: documentos de requisitos)
- [Tipo de artefato 2] (ex: código-fonte)
- [Tipo de artefato 3] (ex: documentação de usuário)
- [Outros artefatos relevantes]

Por favor, ajude-me a criar diretrizes para:

1. Organização e Estrutura:
   - Hierarquia de repositórios/pastas
   - Convenções de nomenclatura
   - Metadados necessários

2. Controle de Versão:
   - Esquema de versionamento
   - Rastreabilidade entre versões
   - Processo de branching para documentos

3. Processos de Aprovação:
   - Fluxo de aprovação por tipo de artefato
   - Níveis de aprovação necessários
   - Registro de aprovações

4. Políticas de Acesso:
   - Níveis de permissão
   - Controle de acesso por perfil
   - Procedimentos de segurança

5. Retenção e Arquivamento:
   - Período de retenção por tipo de artefato
   - Processo de arquivamento
   - Recuperação de artefatos arquivados

6. Ferramentas e Integração:
   - Ferramentas recomendadas por tipo de artefato
   - Integração entre ferramentas
   - Automação de processos

Forneça exemplos concretos e templates sempre que possível, considerando as melhores práticas da indústria.
```
