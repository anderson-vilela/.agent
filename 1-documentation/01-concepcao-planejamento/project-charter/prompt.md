# Prompts para Project Charter

## Prompt para Criação de Project Charter Completo

```
Como gerente de projetos, preciso desenvolver um Project Charter completo para o projeto [NOME DO PROJETO].

Contexto do projeto:
- Descrição breve: [DESCRIÇÃO]
- Objetivo principal: [OBJETIVO]
- Patrocinador: [PATROCINADOR]
- Stakeholders-chave: [STAKEHOLDERS]
- Restrições conhecidas: [RESTRIÇÕES]

Por favor, elabore um Project Charter abrangente e estruturado que inclua:

1. Informações do Projeto
   - Nome do projeto
   - Código ou identificador único
   - Data de início e término previsto
   - Versão do documento e histórico de revisões

2. Propósito e Justificativa
   - Necessidade de negócio que originou o projeto
   - Problema ou oportunidade sendo abordada
   - Alinhamento com objetivos estratégicos
   - Benefícios esperados (tangíveis e intangíveis)
   - Consequências de não realizar o projeto

3. Descrição do Projeto
   - Visão geral do projeto
   - Objetivos específicos (SMART)
   - Principais entregas
   - Critérios de aceitação de alto nível
   - Exclusões explícitas (o que não faz parte do escopo)

4. Requisitos de Alto Nível
   - Requisitos de negócio essenciais
   - Requisitos técnicos fundamentais
   - Requisitos de qualidade críticos
   - Requisitos de conformidade e regulatórios

5. Premissas e Restrições
   - Premissas principais que fundamentam o planejamento
   - Restrições de orçamento
   - Restrições de prazo
   - Restrições de recursos
   - Restrições técnicas ou organizacionais

6. Riscos de Alto Nível
   - Identificação dos principais riscos
   - Avaliação preliminar (probabilidade e impacto)
   - Estratégias iniciais de resposta
   - Potenciais oportunidades (riscos positivos)

7. Marcos e Timeline
   - Marcos principais do projeto
   - Datas-chave
   - Dependências críticas externas
   - Faseamento de alto nível

8. Orçamento Estimado
   - Estimativa de custos de alto nível
   - Fontes de financiamento
   - Abordagem orçamentária
   - Limitações financeiras

9. Stakeholders
   - Identificação dos stakeholders-chave
   - Papéis e interesses
   - Impacto e influência
   - Expectativas principais

10. Estrutura de Governança
    - Patrocinador do projeto
    - Gerente do projeto e responsabilidades
    - Comitê diretivo (se aplicável)
    - Processo de escalação de questões
    - Requisitos de aprovação e reportes

11. Abordagem do Projeto
    - Metodologia a ser utilizada
    - Ciclo de vida do projeto
    - Ferramentas principais
    - Considerações especiais

12. Aprovações
    - Seção para assinaturas dos aprovadores
    - Datas de aprovação
    - Condições para revisão do charter

O documento deve ser claro, conciso e fornecer autoridade formal para o gerente de projeto, estabelecendo os parâmetros iniciais dentro dos quais o projeto operará.
```

## Prompt para Definição de Objetivos SMART

```
Como coach de gerenciamento de projetos, preciso desenvolver objetivos SMART (Específicos, Mensuráveis, Atingíveis, Relevantes e Temporais) para o Project Charter do projeto [NOME DO PROJETO].

Contexto do projeto:
- Finalidade geral: [FINALIDADE]
- Resultados esperados: [RESULTADOS]
- Beneficiários principais: [BENEFICIÁRIOS]
- Prazo geral: [PRAZO]
- Recursos disponíveis: [RECURSOS]

Por favor, formule um conjunto de objetivos SMART que:

1. Transforme Objetivos Genéricos em Específicos
   - Detalhe exatamente o que o projeto busca alcançar
   - Especifique quem é impactado ou beneficiado
   - Indique onde ocorrerá (sistemas, departamentos, locais)
   - Esclareça elementos específicos que definem sucesso

2. Defina Métricas de Mensuração
   - Indicadores quantitativos para cada objetivo
   - Métricas qualitativas quando apropriado
   - Baselines e valores-alvo específicos
   - Métodos de verificação e medição

3. Assegure que são Atingíveis
   - Análise de viabilidade considerando recursos disponíveis
   - Comparação com benchmarks e projetos anteriores
   - Identificação de capacidades necessárias
   - Nível de desafio apropriado (desafiador mas alcançável)

4. Demonstre Relevância
   - Conexão clara com objetivos estratégicos
   - Valor de negócio específico
   - Prioridade relativa entre objetivos
   - Justificativa de por que cada objetivo importa

5. Estabeleça Temporalidade
   - Datas-limite específicas para cada objetivo
   - Marcos intermediários ou fases
   - Sequenciamento lógico quando aplicável
   - Considerações sobre janelas de oportunidade

Para cada objetivo SMART, forneça:
- Formulação clara e direta
- Explicação dos elementos SMART presentes
- Desafios potenciais para alcançá-lo
- Dicas para rastreamento e monitoramento
- Exemplo de como seria reportado o progresso

Os objetivos devem ser independentes mas coesos, coletivamente abrangendo o propósito completo do projeto sem sobreposição ou ambiguidade. A linguagem deve ser clara e precisa, evitando termos vagos como "melhorar", "otimizar" ou "aumentar" sem as respectivas qualificações.
```

## Prompt para Análise de Stakeholders

```
Como especialista em gestão de stakeholders, preciso desenvolver uma análise detalhada de stakeholders para o Project Charter do projeto [NOME DO PROJETO].

Contexto do projeto:
- Escopo geral: [ESCOPO]
- Impacto organizacional: [IMPACTO]
- Áreas envolvidas: [ÁREAS]
- Duração estimada: [DURAÇÃO]
- Complexidade política: [BAIXA/MÉDIA/ALTA]

Por favor, desenvolva uma análise abrangente de stakeholders que inclua:

1. Metodologia de Identificação
   - Abordagem para mapeamento completo de stakeholders
   - Critérios para categorização e priorização
   - Framework para análise e planejamento

2. Mapeamento de Stakeholders Internos
   - Executivos e patrocinadores
   - Gerência funcional impactada
   - Membros da equipe do projeto
   - Departamentos de suporte
   - Usuários finais internos
   - Outros grupos internos afetados

3. Mapeamento de Stakeholders Externos
   - Clientes e usuários externos
   - Fornecedores e parceiros
   - Reguladores e entidades governamentais
   - Público geral ou comunidades
   - Investidores ou financiadores
   - Concorrentes relevantes

4. Análise de Interesse e Influência
   - Matriz de poder/interesse
   - Nível de impacto do projeto sobre cada stakeholder
   - Nível de influência de cada stakeholder sobre o projeto
   - Posicionamento atual (apoiador, neutro, resistente)
   - Razões para apoio ou resistência

5. Necessidades e Expectativas
   - Principais interesses de cada grupo de stakeholders
   - Expectativas específicas sobre resultados
   - Preocupações e possíveis objeções
   - Critérios de satisfação por stakeholder
   - Potenciais conflitos entre expectativas

6. Estratégia de Gestão por Grupo
   - Abordagem de engajamento (informar, consultar, envolver, colaborar)
   - Frequência e métodos de comunicação
   - Nível de participação nas decisões
   - Estratégias para gerenciar resistência
   - Abordagem para gerenciar conflitos entre stakeholders

7. Plano de Engajamento de Alto Nível
   - Momentos-chave para comunicação ou engajamento
   - Responsáveis por relacionamento com cada grupo
   - Mensagens-chave por stakeholder
   - Indicadores de engajamento efetivo
   - Processo para feedback contínuo

8. Matriz RACI para Stakeholders Principais
   - Responsáveis (Responsible)
   - Aprovadores (Accountable)
   - Consultados (Consulted)
   - Informados (Informed)

A análise deve ser estratégica e prática, reconhecendo a natureza dinâmica das relações com stakeholders e fornecendo direcionamento claro para o gerente de projeto sobre como construir e manter o apoio necessário para o sucesso do projeto.
```

## Prompt para Identificação e Análise de Riscos Iniciais

```
Como especialista em gestão de riscos, preciso desenvolver uma análise inicial de riscos para o Project Charter do projeto [NOME DO PROJETO].

Contexto do projeto:
- Tipo de projeto: [TIPO]
- Objetivos principais: [OBJETIVOS]
- Complexidade estimada: [COMPLEXIDADE]
- Inovação envolvida: [NÍVEL DE INOVAÇÃO]
- Restrições críticas: [RESTRIÇÕES]

Por favor, elabore uma análise preliminar de riscos que inclua:

1. Metodologia de Identificação
   - Técnicas utilizadas para identificação de riscos
   - Categorias de riscos consideradas
   - Fontes de informação consultadas
   - Limitações da análise inicial

2. Registro de Riscos de Alto Nível
   - ID único e nome descritivo para cada risco
   - Descrição clara (causa, evento de risco, impacto)
   - Categorização (técnico, externo, organizacional, gerencial)
   - Probabilidade inicial (escala definida)
   - Impacto potencial (escala definida)
   - Severidade (combinação de probabilidade e impacto)
   - Gatilhos ou sinais de alerta

3. Riscos Técnicos
   - Complexidade tecnológica
   - Requisitos indefinidos ou instáveis
   - Dependências técnicas
   - Interoperabilidade e integração
   - Performance e escalabilidade
   - Segurança e privacidade

4. Riscos Externos
   - Mudanças regulatórias ou legais
   - Condições de mercado ou econômicas
   - Fornecedores e terceiros
   - Fatores ambientais ou sociais
   - Concorrência e ambiente competitivo
   - Eventos imprevistos (desastres naturais, crises)

5. Riscos Organizacionais
   - Disponibilidade de recursos
   - Competências necessárias
   - Estrutura organizacional e governança
   - Cultura e resistência à mudança
   - Conflitos de prioridade
   - Suporte gerencial e patrocínio

6. Riscos de Gerenciamento
   - Estimativas de prazo e orçamento
   - Comunicação e engajamento de stakeholders
   - Planejamento e controle
   - Tomada de decisão
   - Coordenação e interfaces
   - Gestão de mudanças

7. Oportunidades (Riscos Positivos)
   - Potenciais benefícios não planejados
   - Aceleradores de cronograma
   - Eficiências potenciais
   - Sinergias possíveis
   - Inovações emergentes

8. Estratégias Iniciais de Resposta
   - Abordagens preliminares para riscos prioritários
   - Potenciais ações preventivas
   - Planos de contingência para riscos críticos
   - Gatilhos para ativação de respostas
   - Responsabilidades iniciais

9. Recomendações para Planejamento
   - Riscos prioritários para monitoramento imediato
   - Análises adicionais necessárias
   - Sugestões para o Plano de Gerenciamento de Riscos
   - Reservas de contingência recomendadas
   - Frequência de reavaliação sugerida

A análise deve ser equilibrada, identificando ameaças e oportunidades relevantes, com foco nos riscos que poderiam significativamente impactar o sucesso do projeto, permitindo decisões informadas antes de prosseguir para o planejamento detalhado.
```

## Prompt para Desenvolvimento de Critérios de Sucesso

```
Como consultor de gestão de projetos, preciso definir critérios de sucesso claros para o Project Charter do projeto [NOME DO PROJETO].

Contexto do projeto:
- Propósito principal: [PROPÓSITO]
- Entregáveis principais: [ENTREGÁVEIS]
- Expectativas do patrocinador: [EXPECTATIVAS]
- Restrições críticas: [RESTRIÇÕES]
- Stakeholders-chave: [STAKEHOLDERS]

Por favor, desenvolva um conjunto abrangente de critérios de sucesso que inclua:

1. Critérios Relacionados a Entregáveis
   - Requisitos fundamentais que devem ser atendidos
   - Especificações técnicas ou de qualidade essenciais
   - Standards ou normas aplicáveis
   - Critérios de aceitação mensuráveis
   - Processos de validação e verificação

2. Critérios de Desempenho do Projeto
   - Métricas de prazo (tolerâncias e limites)
   - Métricas de orçamento (tolerâncias e limites)
   - Métricas de escopo (completude e estabilidade)
   - Métricas de qualidade (defeitos, retrabalho)
   - Métricas de recursos (utilização, produtividade)

3. Critérios de Impacto de Negócio
   - ROI ou métricas financeiras específicas
   - Benefícios quantificáveis esperados
   - Indicadores de performance operacional
   - Melhoria em KPIs organizacionais
   - Contribuições para objetivos estratégicos

4. Critérios de Satisfação de Stakeholders
   - Métricas de satisfação do cliente/usuário
   - Expectativas específicas de stakeholders-chave
   - Percepção de valor e utilidade
   - Adoção e utilização
   - Feedback qualitativo

5. Critérios de Transição e Implementação
   - Critérios para transição para operações
   - Capacitação e transferência de conhecimento
   - Documentação necessária
   - Estabilidade pós-implementação
   - Suporte inicial e resolução de problemas

6. Definição Estruturada de Sucesso
   - Critérios mínimos (must-have para o projeto ser considerado bem-sucedido)
   - Critérios desejáveis (should-have para otimizar valor)
   - Critérios ampliados (could-have para exceder expectativas)
   - Priorização e pesos relativos
   - Interrelações e dependências entre critérios

7. Métodos de Medição e Avaliação
   - Como cada critério será medido
   - Fontes de dados e evidências
   - Frequência de avaliação
   - Responsáveis pela medição
   - Processo de reportes e comunicação de progresso

8. Processo de Revisão e Adaptação
   - Gatilhos para revisão de critérios
   - Processo para ajustes durante o projeto
   - Quando e como reavaliar expectativas
   - Gestão de divergências sobre sucesso
   - Lições aprendidas sobre definição de sucesso

Os critérios devem ser específicos, mensuráveis, consensuais e realistas, fornecendo uma base clara para avaliar o sucesso do projeto tanto durante sua execução quanto após sua conclusão. Eles devem também considerar diferentes perspectivas de sucesso (técnica, comercial, organizacional, stakeholders), criando uma visão balanceada que alinhe expectativas diversas.
```
