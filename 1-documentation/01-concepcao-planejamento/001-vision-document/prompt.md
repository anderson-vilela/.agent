# vision-document

# Prompts para Vision Document

## Prompt para Criação de Vision Document Completo

```
Como arquiteto de soluções, preciso criar um Vision Document abrangente para o projeto [NOME DO PROJETO].

Contexto do projeto:
- Descrição: [BREVE DESCRIÇÃO]
- Problema central: [PROBLEMA QUE O PROJETO RESOLVE]
- Público-alvo: [QUEM UTILIZARÁ O SISTEMA]
- Tecnologias previstas: [TECNOLOGIAS CONSIDERADAS]

Por favor, crie um Vision Document completo e estruturado que inclua:

1. Introdução
   - Propósito do documento
   - Escopo do projeto
   - Definições, acrônimos e abreviações
   - Referências e documentos relacionados

2. Posicionamento
   - Oportunidade de negócio
   - Descrição do problema
   - Posicionamento do produto
   - Stakeholders e perfis de usuário

3. Visão geral do produto
   - Perspectiva do produto
   - Resumo das capacidades
   - Suposições e dependências
   - Alternativas e competição

4. Features e requisitos de alto nível
   - Features principais
   - Requisitos funcionais críticos
   - Requisitos não-funcionais essenciais
   - Restrições de design e implementação

5. Restrições e riscos
   - Limitações conhecidas
   - Riscos de alto nível
   - Mitigações propostas

O documento deve equilibrar inspiração e realismo, oferecendo uma visão clara sem se perder em detalhes prematuros de implementação.
```

## Prompt para Seção de Análise de Stakeholders

```
Como analista de negócios, preciso elaborar a seção de análise de stakeholders para o Vision Document do projeto [NOME DO PROJETO].

Contexto do projeto:
- Objetivo principal: [OBJETIVO]
- Área de negócio: [ÁREA]
- Impacto organizacional: [IMPACTO]

Por favor, desenvolva uma análise completa de stakeholders que inclua:

1. Identificação de stakeholders
   - Lista categorizada (internos/externos, primários/secundários)
   - Papéis e responsabilidades
   - Influência e interesse no projeto

2. Perfis detalhados dos stakeholders principais
   - Necessidades e expectativas
   - Preocupações e resistências potenciais
   - Nível de envolvimento desejado
   - Impacto do projeto em suas atividades

3. Perfis de usuário final
   - Personas detalhadas
   - Necessidades e objetivos
   - Contexto de uso
   - Experiência técnica

4. Estratégia de gerenciamento de stakeholders
   - Abordagem de comunicação por grupo
   - Frequência e métodos de engajamento
   - Estratégias para lidar com expectativas conflitantes

A análise deve ser detalhada o suficiente para guiar o desenvolvimento, mas concisa o bastante para ser facilmente compreendida por todos os envolvidos.
```

## Prompt para Escopo e Não-Escopo

```
Como gerente de produto, preciso definir claramente o que está dentro e fora do escopo para o Vision Document do projeto [NOME DO PROJETO].

Contexto do projeto:
- Objetivos principais: [OBJETIVOS]
- Restrições: [RESTRIÇÕES]
- Timeline: [TIMELINE]
- Recursos disponíveis: [RECURSOS]

Por favor, crie uma definição clara e objetiva de escopo e não-escopo que inclua:

1. Escopo do produto
   - Funcionalidades principais (MVP)
   - Capacidades secundárias (prioridade menor)
   - Integrações necessárias
   - Plataformas suportadas
   - Dados gerenciados

2. Escopo do projeto
   - Fases planejadas
   - Entregas principais
   - Cronograma de alto nível
   - Recursos e equipes envolvidas
   - Critérios de aceitação principais

3. Explicitamente fora do escopo
   - Funcionalidades descartadas ou adiadas
   - Integrações não contempladas
   - Limitações intencionais
   - Plataformas não suportadas
   - Casos de uso não atendidos

4. Critérios para mudanças de escopo
   - Processo para avaliar mudanças
   - Thresholds para reavaliação
   - Impacto admissível em cronograma/orçamento

A definição deve ser inequívoca, evitando ambiguidades que possam levar a expectativas desalinhadas ou scope creep.
```

## Prompt para Métricas de Sucesso

```
Como consultor estratégico, preciso definir métricas claras de sucesso para o Vision Document do projeto [NOME DO PROJETO].

Contexto do projeto:
- Objetivos de negócio: [OBJETIVOS]
- Problema resolvido: [PROBLEMA]
- Público-alvo: [PÚBLICO]
- Proposição de valor: [VALOR]

Por favor, elabore um conjunto abrangente de métricas de sucesso que inclua:

1. Métricas de negócio
   - KPIs financeiros (ROI, receita, economia de custos)
   - KPIs de crescimento
   - Métricas de mercado
   - Indicadores de eficiência operacional

2. Métricas de produto
   - Adoção e engajamento
   - Performance técnica
   - Qualidade e confiabilidade
   - Experiência do usuário

3. Métricas de projeto
   - Cumprimento de cronograma
   - Aderência ao orçamento
   - Qualidade das entregas
   - Eficácia da gestão de riscos

4. Framework de mensuração
   - Métodos de coleta de dados
   - Frequência de avaliação
   - Valores de baseline
   - Metas para diferentes horizontes (curto, médio, longo prazo)
   - Responsáveis pelo tracking

As métricas devem ser SMART (Específicas, Mensuráveis, Atingíveis, Relevantes e Temporais) e diretamente ligadas aos objetivos estratégicos do projeto.
```

## Prompt para Visão de Produto Inspiradora

```
Como visionário de produto, preciso criar uma declaração de visão inspiradora mas realista para o Vision Document do projeto [NOME DO PROJETO].

Contexto do projeto:
- Inovação proposta: [INOVAÇÃO]
- Diferencial competitivo: [DIFERENCIAL]
- Estado atual: [ESTADO ATUAL]
- Estado futuro desejado: [ESTADO FUTURO]

Por favor, desenvolva uma visão de produto completa e inspiradora que inclua:

1. Declaração de visão concisa
   - Uma frase poderosa que captura a essência
   - "Para [PÚBLICO-ALVO], o [PRODUTO] é um [CATEGORIA] que [BENEFÍCIO PRINCIPAL]. Diferente de [ALTERNATIVA], nosso produto [DIFERENCIADOR CHAVE]."

2. Narrativa expandida
   - Descrição vívida do futuro com o produto implementado
   - Impacto na vida dos usuários
   - Transformação proporcionada ao negócio
   - Contribuição para o mercado/sociedade

3. Princípios norteadores
   - Valores fundamentais que guiam decisões
   - Compromissos inegociáveis
   - Filosofia de design
   - Prioridades estratégicas

4. Evolução da visão
   - Visão imediata (1 ano)
   - Visão de médio prazo (2-3 anos)
   - Visão de longo prazo (5+ anos)

A visão deve ser ambiciosa o suficiente para inspirar e orientar, mas realista o suficiente para ser credível e acionável.
```
