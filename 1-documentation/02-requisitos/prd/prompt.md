# Prompts para Product Requirements Document (PRD)

## Prompt para Estrutura Completa do PRD

```
Você é um Product Manager experiente com profundo conhecimento em documentação de requisitos de produto. Preciso da sua ajuda para criar um Product Requirements Document (PRD) abrangente para [nome do produto/projeto].

Contexto do produto:
- Descrição: [breve descrição do produto]
- Público-alvo: [descrição dos usuários-alvo]
- Problema a ser resolvido: [problema principal que o produto visa resolver]
- Objetivos de negócio: [principais objetivos de negócio]

Por favor, crie um PRD completo seguindo as melhores práticas do setor, incluindo todas as seções necessárias como visão geral do produto, declaração do problema, público-alvo, objetivos e métricas de sucesso, funcionalidades (organizadas por prioridade), restrições, premissas e dependências.

Para cada funcionalidade, inclua:
- Nome e descrição clara
- Prioridade (usando método MoSCoW: Must-have, Should-have, Could-have, Won't-have)
- Critérios de aceitação mensuráveis
- Comportamento esperado
- Possíveis casos de borda ou considerações especiais
```

## Prompt para Definição de Funcionalidades

```
Como Product Manager, preciso definir detalhadamente as funcionalidades para [nome da funcionalidade] no nosso PRD.

Contexto:
- Produto: [nome do produto]
- Usuários-alvo desta funcionalidade: [quem utilizará]
- Problema específico que esta funcionalidade resolve: [descrição do problema]

Para cada funcionalidade abaixo, ajude-me a criar uma definição completa incluindo:
1. Descrição detalhada
2. Comportamento esperado
3. Critérios de aceitação específicos e mensuráveis
4. Prioridade (Must, Should, Could, Won't)
5. Dependências
6. Considerações de UX
7. Possíveis extensões futuras

Funcionalidades para definir:
- [Funcionalidade 1]
- [Funcionalidade 2]
- [Funcionalidade 3]
```

## Prompt para Métricas de Sucesso do Produto

```
Como especialista em produto, preciso definir métricas de sucesso claras e mensuráveis para [nome do produto] que serão incluídas no PRD.

Contexto:
- Visão do produto: [visão]
- Objetivos de negócio: [objetivos]
- Comportamento atual dos usuários (se aplicável): [comportamentos]
- Desafios atuais: [desafios]

Por favor, ajude-me a definir:

1. KPIs (Key Performance Indicators) relevantes para medir o sucesso do produto
2. Métricas de engajamento do usuário
3. Métricas de valor de negócio
4. Métricas de qualidade do produto
5. Como e quando essas métricas devem ser medidas
6. Valores de referência (benchmarks) ou metas específicas para cada métrica

Explique a relevância de cada métrica proposta em relação aos objetivos do produto.
```

## Prompt para Análise de Competidores e Diferenciação

```
Como analista de produto, preciso incluir no PRD uma análise competitiva para [nome do produto], identificando como nos diferenciaremos no mercado.

Os principais competidores são:
- [Competidor 1]
- [Competidor 2]
- [Competidor 3]

Para cada competidor, por favor ajude-me a analisar:
1. Principais recursos e funcionalidades
2. Pontos fortes e fracos
3. Estratégia de preços e modelo de negócio
4. Público-alvo e posicionamento de mercado
5. Experiência do usuário

Com base nessa análise, ajude-me a elaborar:
1. Uma tabela comparativa de recursos
2. Proposta de valor único do nosso produto
3. Oportunidades de diferenciação
4. Potenciais riscos ou ameaças competitivas
5. Estratégias recomendadas para diferenciação
```

## Prompt para Roadmap Inicial Baseado no PRD

```
Como Product Manager, preciso desenvolver um roadmap inicial baseado no PRD para [nome do produto]. O horizonte de tempo a ser considerado é de [período, ex: 12 meses].

Considerando as seguintes informações do nosso PRD:
- Funcionalidades prioritárias: [listar funcionalidades Must-have e Should-have]
- Recursos disponíveis: [informações sobre equipe, restrições, etc.]
- Dependências conhecidas: [listar dependências]
- Objetivos de negócio: [objetivos principais]

Por favor, ajude-me a criar:

1. Um roadmap de alto nível dividido em fases ou releases
2. Recomendações de priorização para as primeiras releases
3. Marcos (milestones) importantes
4. Dependências críticas entre funcionalidades
5. Sugestões de MVPs (Minimum Viable Products) potenciais
6. Considerações sobre trade-offs entre escopo, tempo e recursos
```
