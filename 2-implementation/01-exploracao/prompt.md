# Prompt para Fase de Exploração

## Contexto

Estou na fase inicial de exploração para implementação de uma nova funcionalidade/componente. Preciso realizar uma investigação completa antes de começar a programação efetiva.

## Objetivo

Utilize sua capacidade de análise e conhecimento técnico para me ajudar a explorar:

- Requisitos e documentação técnica
- Código existente relevante
- Soluções potenciais e alternativas
- Provas de conceito

## Prompts por Etapa

### 1. Revisão da Documentação Técnica

```
Estou revisando a documentação técnica para implementar [DESCRIÇÃO DA FUNCIONALIDADE].
Dados os seguintes requisitos e informações de arquitetura:

[INSERIR REQUISITOS FUNCIONAIS]
[INSERIR REQUISITOS NÃO-FUNCIONAIS]
[INSERIR INFORMAÇÕES DE ARQUITETURA]

Ajude-me a:
1. Identificar os principais componentes e interfaces que serão afetados
2. Destacar possíveis desafios técnicos e restrições
3. Sugerir perguntas importantes que preciso responder antes de prosseguir
4. Recomendar áreas da documentação que devo explorar mais profundamente
```

### 2. Análise do Código Existente

```
Estou analisando o código existente relacionado a [COMPONENTE/FUNCIONALIDADE].
Aqui estão as partes relevantes do código:

[INSERIR TRECHOS DE CÓDIGO RELEVANTES]

Ajude-me a:
1. Entender o fluxo e a lógica principal deste código
2. Identificar padrões de design e convenções utilizadas
3. Mapear dependências e integrações importantes
4. Apontar áreas que precisarão ser modificadas ou estendidas
5. Destacar potenciais problemas ou desafios na implementação
```

### 3. Pesquisa de Soluções

```
Estou pesquisando soluções para implementar [DESCRIÇÃO DO PROBLEMA/FUNCIONALIDADE].
O contexto é:

[INSERIR CONTEXTO DO PROJETO]
[INSERIR REQUISITOS ESPECÍFICOS]
[INSERIR RESTRIÇÕES TECNOLÓGICAS]

Ajude-me a:
1. Identificar bibliotecas e frameworks que poderiam ser utilizados
2. Comparar diferentes abordagens técnicas possíveis
3. Analisar prós e contras de cada solução
4. Sugerir padrões de design aplicáveis
5. Recomendar recursos ou documentação para aprofundar meu conhecimento
```

### 4. Prototipagem de Conceitos

```
Quero criar uma prova de conceito para [DESCRIÇÃO DA FUNCIONALIDADE].
Preciso validar:

[INSERIR ASPECTOS TÉCNICOS A SEREM VALIDADOS]
[INSERIR PREOCUPAÇÕES DE PERFORMANCE/SEGURANÇA/ETC]

Ajude-me a:
1. Desenvolver um esboço de código para este protótipo
2. Focar nos aspectos críticos que precisam ser validados
3. Simular cenários relevantes para testar a abordagem
4. Identificar métricas para avaliar o sucesso do protótipo
5. Sugerir simplificações que mantenham a validade do teste
```

## Prompts para Casos Específicos

### Análise de Complexidade Algorítmica

```
Preciso entender a complexidade e eficiência de diferentes algoritmos para [PROBLEMA ESPECÍFICO].
As opções que estou considerando são:

[INSERIR ALGORITMO 1]
[INSERIR ALGORITMO 2]
[INSERIR ALGORITMO 3]

Ajude-me a:
1. Analisar a complexidade computacional (tempo e espaço) de cada opção
2. Identificar trade-offs entre as diferentes abordagens
3. Recomendar a melhor solução considerando o contexto e escala de [PARÂMETROS RELEVANTES]
```

### Avaliação de Arquitetura

```
Estou avaliando diferentes arquiteturas para implementar [COMPONENTE/SISTEMA].
As opções que estou considerando são:

[INSERIR ARQUITETURA 1]
[INSERIR ARQUITETURA 2]
[INSERIR ARQUITETURA 3]

Considerando os requisitos:
- [REQUISITO 1]
- [REQUISITO 2]
- [REQUISITO 3]

Ajude-me a:
1. Analisar como cada arquitetura atende aos requisitos
2. Identificar pontos fortes e fracos de cada abordagem
3. Avaliar aspectos como escalabilidade, manutenibilidade e performance
4. Recomendar a arquitetura mais adequada com justificativa
```

### Exploração de Tecnologias

```
Preciso escolher uma tecnologia para [FINALIDADE ESPECÍFICA].
As opções que estou considerando são:

[INSERIR TECNOLOGIA 1]
[INSERIR TECNOLOGIA 2]
[INSERIR TECNOLOGIA 3]

Nosso contexto é:
- [INFORMAÇÃO DE CONTEXTO 1]
- [INFORMAÇÃO DE CONTEXTO 2]
- [INFORMAÇÃO DE CONTEXTO 3]

Ajude-me a:
1. Comparar as tecnologias em termos de recursos, comunidade, maturidade e documentação
2. Analisar a adequação de cada uma para nosso contexto específico
3. Identificar possíveis limitações ou desafios com cada opção
4. Recomendar a melhor opção com justificativa
```

## Dicas para Usar Este Prompt

1. **Seja específico**: Quanto mais detalhes você fornecer sobre o contexto, requisitos e código existente, melhores serão as respostas.

2. **Divida e conquiste**: Para sistemas complexos, considere explorar um componente ou aspecto por vez.

3. **Itere**: Use as respostas para refinar suas perguntas e aprofundar áreas específicas.

4. **Questione**: Peça ao Copilot para explicar o raciocínio por trás das sugestões e para propor alternativas.

5. **Valide**: Sempre avalie criticamente as sugestões recebidas, especialmente para decisões arquiteturais importantes.
