# Prompts para Definição do Escopo de Implementação

## Definição Inicial do Escopo

```
Como Arquiteto de Software experiente, preciso da sua ajuda para definir claramente o escopo da seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Por favor, ajude-me a:

1. Criar uma lista clara do que ESTÁ incluído no escopo desta implementação
2. Criar uma lista explícita do que NÃO está incluído no escopo desta implementação
3. Identificar todas as interfaces necessárias com outros componentes do sistema
4. Estabelecer claramente as fronteiras e responsabilidades deste módulo
5. Documentar quaisquer pressupostos ou restrições técnicas importantes

Estruture sua resposta de forma que possa ser facilmente compartilhada com a equipe de desenvolvimento e stakeholders.
```

## Visualização do Escopo com Diagrama

```
Como Arquiteto de Sistemas especializado em modelagem, preciso da sua ajuda para criar um diagrama conceitual que visualize claramente o escopo da seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Com base nas seguintes informações de escopo:

[INCLUA AQUI AS INFORMAÇÕES BÁSICAS DE ESCOPO JÁ DEFINIDAS]

Por favor, descreva um diagrama de contexto ou componentes que mostre:

1. O componente/módulo que estamos implementando (fronteiras claras)
2. Interfaces e pontos de integração com outros componentes do sistema
3. Fluxos de dados ou interações principais
4. Dependências externas relevantes

Descreva o diagrama em texto de forma que eu possa recriá-lo em uma ferramenta de diagramação, incluindo os elementos, suas relações e qualquer informação importante sobre como o diagrama deve ser estruturado.
```

## Matriz de Responsabilidades

```
Como Product Manager com experiência em delineamento de responsabilidades, preciso da sua ajuda para criar uma matriz clara de responsabilidades para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Baseado nas seguintes informações:

[INCLUA AQUI AS INFORMAÇÕES BÁSICAS DE ESCOPO JÁ DEFINIDAS]

Por favor, crie uma matriz RACI (Responsible, Accountable, Consulted, Informed) ou similar que:

1. Liste todas as funcionalidades/capacidades principais identificadas no escopo
2. Identifique claramente qual componente/sistema é responsável por cada funcionalidade
3. Especifique quando a responsabilidade é compartilhada entre múltiplos componentes
4. Destaque áreas onde há potencial ambiguidade de responsabilidade

Formate a matriz de forma que seja facilmente compreensível pela equipe técnica e de produto.
```

## Definição de Critérios de Conclusão

```
Como Tech Lead experiente, preciso da sua ajuda para definir critérios claros de conclusão para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Com base no escopo definido:

[INCLUA AQUI AS INFORMAÇÕES BÁSICAS DE ESCOPO JÁ DEFINIDAS]

Por favor, ajude-me a criar:

1. Uma lista de critérios objetivos e verificáveis que indicarão que a implementação foi concluída com sucesso
2. Métricas técnicas ou de qualidade que devem ser alcançadas (cobertura de testes, performance, etc.)
3. Entregáveis específicos que devem ser produzidos (código, documentação, etc.)
4. Condições que definirão quando o trabalho poderá ser considerado "pronto"

O objetivo é ter critérios claros que possam ser usados por toda a equipe para avaliar quando o trabalho de implementação estiver completo.
```

## Análise de Riscos de Escopo

```
Como Gerente de Projetos com expertise em análise de riscos, preciso da sua ajuda para identificar potenciais riscos relacionados ao escopo da seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Com base na definição de escopo:

[INCLUA AQUI AS INFORMAÇÕES BÁSICAS DE ESCOPO JÁ DEFINIDAS]

Por favor, forneça:

1. Uma lista dos principais riscos relacionados ao escopo (expansão não controlada, ambiguidades, etc.)
2. Avaliação de probabilidade e impacto para cada risco identificado
3. Estratégias de mitigação para cada risco
4. Sinais de alerta que indicariam que um risco está se materializando

Estruture a análise em formato de matriz de riscos, com probabilidade, impacto e estratégias claramente definidos.
```

## Documento de Escopo Consolidado

```
Como Analista de Negócios especializado em documentação técnica, preciso da sua ajuda para criar um documento consolidado de escopo para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Utilizando todas as informações que temos sobre o escopo:

[INCLUA AQUI TODAS AS INFORMAÇÕES COMPILADAS SOBRE O ESCOPO]

Por favor, crie um documento estruturado que:

1. Comece com um sumário executivo claro e conciso
2. Detalhe o que está dentro e fora do escopo
3. Descreva interfaces e integrações
4. Especifique pressupostos e restrições
5. Apresente critérios de aceitação e conclusão
6. Inclua referências a outros documentos relevantes

O documento deve ser estruturado de forma profissional, com seções claramente delimitadas, formatação consistente e linguagem precisa.
```
