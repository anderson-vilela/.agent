# Prompt para Contextualização

## Definição do Escopo de Implementação

```
Como Arquiteto de Software experiente, me ajude a definir claramente o escopo da seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Por favor, ajude-me a:

1. Delimitar precisamente o que está incluído e o que está fora do escopo desta implementação
2. Identificar as interfaces necessárias com outros componentes do sistema
3. Estabelecer claramente as fronteiras e responsabilidades deste módulo
4. Documentar quaisquer pressupostos ou restrições técnicas importantes

Inclua diagramas conceituais se achar útil para esclarecer as fronteiras do escopo.
```

## Compreensão do Contexto de Negócio

```
Como Analista de Negócios com vasta experiência em traduzir necessidades de negócio em requisitos técnicos, me ajude a compreender o contexto de negócio para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Por favor, elabore sobre:

1. O valor de negócio específico que esta implementação trará para a organização e usuários
2. As regras de negócio críticas que precisam ser incorporadas ou respeitadas
3. Como os requisitos técnicos se alinham com os objetivos estratégicos do negócio
4. Métricas de sucesso do ponto de vista de negócio para esta implementação
5. Quaisquer considerações regulatórias ou de conformidade relevantes

Organize sua resposta de forma que possa ser facilmente compartilhada com stakeholders técnicos e não-técnicos.
```

## Análise de Impacto

```
Como Arquiteto de Sistemas especializado em gestão de mudanças e integração, me ajude a realizar uma análise de impacto completa para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Por favor, forneça uma análise detalhada sobre:

1. Quais áreas ou componentes do sistema atual serão diretamente ou indiretamente afetados
2. Potenciais riscos de integração e como mitigá-los
3. Considerações sobre backward compatibility e estratégias para garantir transição suave
4. Impactos em performance, segurança, escalabilidade ou outras propriedades não-funcionais
5. Dependências externas que podem ser afetadas ou precisam ser consideradas

Estruture sua análise em forma de matriz de impacto com níveis de severidade e probabilidade.
```

## Alinhamento com Stakeholders

```
Como Gerente de Produto experiente com forte background técnico, me ajude a preparar um plano de alinhamento com stakeholders para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Por favor, elabore sobre:

1. Quais stakeholders chave precisam ser consultados e mantidos informados
2. Pontos específicos que precisam ser validados com o Product Owner
3. Expectativas a serem alinhadas com a equipe de QA/Testes
4. Abordagens técnicas que devem ser discutidas com arquitetos e tech leads
5. Um formato eficaz para documentar e comunicar as decisões tomadas

Inclua um modelo para documentação de decisões técnicas (lightweight ADR) que possa ser usado para registrar os alinhamentos realizados.
```

## Documentação de Contextualização Completa

```
Como Technical Writer especializado em documentação de software, me ajude a consolidar a contextualização completa para a seguinte implementação:

[DESCREVA BREVEMENTE A FUNCIONALIDADE/COMPONENTE A SER IMPLEMENTADO]

Com base nas informações coletadas nas etapas anteriores (escopo, contexto de negócio, análise de impacto e alinhamento com stakeholders), crie um documento consolidado de contextualização que:

1. Resume de forma clara o escopo e as fronteiras da implementação
2. Articula o valor de negócio e as regras críticas a serem consideradas
3. Apresenta visualmente as áreas impactadas e os riscos identificados
4. Documenta os alinhamentos realizados com stakeholders chave
5. Serve como referência clara para as próximas etapas do processo de implementação

Formate o documento de maneira profissional, com seções bem definidas, sumário executivo e apêndices conforme necessário.
```
